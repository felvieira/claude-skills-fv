# Image Generation Pattern

## Uso ideal

- hero image, background, variacoes de assets, edicao de imagem, produto, mock visual

## Fluxo recomendado

1. ler `repo-audit/current.md` e `repo-audit/assets.md` quando existirem
2. verificar identidade visual, paleta e assets existentes
3. decidir entre gerar do zero e editar asset existente
4. escolher provider/modelo por caso de uso
5. registrar prompt, modelo, output e referencia visual usada

## Regra importante

Nao tratar geracao de imagem do app como folha em branco quando o produto ja tiver contexto visual.

# Asset Audit
- Ultima revisao: 2026-03-10
- Logos e marcas: nao ha brand pack formal dentro do repo; o principal ativo visual hoje e a propria estrutura documental do kit
- Icones e favicons: nao ha conjunto consolidado de icones/favicons do produto neste repo; a skill `Image Generator` cobre esse caso quando o projeto consumidor precisar
- Ilustracoes/backgrounds/mascotes: nao ha biblioteca visual consolidada no repo; por isso `Asset Librarian` e `Image Generator` precisam ler primeiro o projeto consumidor
- Fontes: nao ha sistema tipografico fechado neste repo; as regras visuais vivem mais como direcao do que como asset pronto
- Tokens visuais: a referencia de design e mais processual do que estetica; o repo orienta coerencia, nao entrega um design system visual completo
- Riscos de consistencia: qualquer projeto consumidor que ja tenha branding, logos ou paleta precisa ser auditado antes de usar `Image Generator`, para evitar assets destoantes

# Eval - Orchestrator Routing

## Objetivo
Validar que o Orchestrator escolhe pipeline adequado sem inflar contexto.

## Entrada
Um pedido de bugfix simples e um pedido de feature completa.

## Esperado
- pipeline minimo para bugfix
- pipeline completo ou adaptado para feature
- handoff curto seguindo `templates/plan.md`

## Evidencias Minimas
- tipo de task classificado
- skills puladas justificadas
- proxima etapa definida

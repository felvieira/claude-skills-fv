# Stack Flexibility Policy

## Objetivo
Evitar que o kit imponha uma stack fixa quando o repositorio usa outra.

## Principio
A stack de referencia do kit e preferencial, nao obrigatoria.

## Regras
- adaptar recomendacoes ao stack real do projeto
- nao forcar Next.js, Prisma, Zustand, React Query ou Tailwind se o projeto nao usa isso
- separar boa pratica universal de preferencia tecnica
- explicar quando uma recomendacao depende da stack de referencia

## Aplicacao
- manter exemplos da stack de referencia quando ajudam
- deixar claro quando sao exemplos e nao requisito
- priorizar compatibilidade com o repositorio analisado
